The Vision

A physical robot that roams the world, sees its surroundings through a camera, and narrates observations like a knowledgeable travel companion. GPS-aware. Voice-enabled. Knows when to speak and when to stay quiet. Part rover, part drone launcher — built by Irwin, powered by Alzar.

2026-03-10 Done
🎤 Voice Home Assistant
Alzar can now be woken by voice and respond conversationally.
  • Wake word: "Hey Jarvis" via OpenWakeWord (hey_jarvis_v0.1.onnx)
  • Speech-to-text: OpenAI Whisper API
  • Brain: OpenClaw /v1/chat/completions endpoint (session: voice-home)
  • Response spoken via ElevenLabs → ffplay → Aiwa speaker
  • Persistent conversation context across calls
  • Custom wake word path: train via Google Colab or swap in Picovoice Porcupine
2026-03-12 Done
🟦 Kinect Dual-Mode (Stationary / Moving)
RGB processing is gated by robot motion state — reduces noise and compute while moving.
  • Stationary mode — RGB frames delivered normally; vision AI can process
  • Moving mode — RGB processing disabled; feed shows "RGB Paused" overlay
  • Toggle via dashboard buttons in the Kinect feed bar
  • State broadcast over WebSocket; live dot dims when paused
2026-03-13 Done
📡 GPS Integration — Huantian BU-353N5
USB GPS/BeiDou receiver connected to the Pi. Real coordinates now feed the Leaflet map live.
  • Huantian BU-353N5 on /dev/ttyUSB0 at 4800 baud
  • NMEA sentences parsed via pynmea2 in a background thread (modules/gps.py)
  • Map marker updates in real-time via WebSocket state_update
  • GPS status bar shows: fix state, coordinates, ±accuracy, satellite count
  • Defaults to Sydney coordinates while acquiring fix indoors
2026-03-13 Done
🖥️ Dashboard Layout & Kinect Feed Fixes
Live fixes to the dashboard based on real-world use.
  • Kinect feed now streams truly live frames — generator uses threading.Event to yield only on new frames, eliminating freeze on stationary mode
  • Kinect stream reloads automatically when switching back to Stationary mode
  • Kinect feed resized to match Brio camera feed height (from fixed 160px to flex: 1)
  • Map and Controls panels swapped — controls top-right, map bottom-right
  • Grid rows adjusted to 2fr 1fr — more space for cameras, smaller commentary box
2026-03-28 Done
📷 Arducam OwlSight 64MP — CSI Camera Upgrade
Replaced the USB Logitech BRIO with the Arducam OwlSight 64MP (OV64A40) on the Pi 5's CSI CAM1 port.
  • Resolution upgraded from 1280×720 (USB) to 1920×1080 @ 30fps (CSI) — up to 64MP for stills
  • Camera driver: dtoverlay=ov64a40,link-frequency=360000000 in /boot/firmware/config.txt
  • Streaming via rpicam-vid subprocess (MJPEG stdout) — avoids Python version conflicts with libcamera bindings
  • Dashboard feed, vision AI loop, and camera API all preserved — no interface changes required
